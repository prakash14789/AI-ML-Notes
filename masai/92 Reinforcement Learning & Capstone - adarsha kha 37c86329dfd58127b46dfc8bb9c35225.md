## 2. Detailed Explanation

### Reinforcement learning vs supervised and unsupervised learning

Before diving into RL, it helps to see exactly how it differs from the approaches you may already know.

**Supervised learning** trains a model on labelled data — both input features and target labels are provided. Given the physical features of an animal (weight, height, number of legs), the model is told which species each sample belongs to. Because the expected output is always given, this is described as "guided learning". Classification and regression are typically one-step: supply an input, receive an output.

**Unsupervised learning** supplies only inputs — no labels or target outputs. The algorithm finds hidden structure, such as groupings, on its own. Clustering is a canonical example: the algorithm groups similar animals together without ever being told the species names. Like supervised learning, clustering is also a one-step process.

**Reinforcement learning** differs fundamentally from both:

- There is no fixed labelled dataset.
- Learning happens through **interaction with an environment** over time.
- The goal is to find the **best sequence of actions** to accomplish a task — not to label or cluster static data.
- Learning is a **trial-and-error** process: the agent takes random actions at first and improves as it accumulates experience.
- Data is **dynamic**, generated by the agent's own behaviour, not collected in advance.
- Tasks are **sequential** — multiple steps are required, unlike the one-step input-output pattern of supervised learning.

Think of someone dropped in an unfamiliar location. They learn which foods are safe, which areas are dangerous, and how to navigate — not from labelled instructions, but by interacting with the environment and experiencing positive and negative outcomes. Eating a bitter fruit is a negative reward; finding good food is a positive reward.

### The core components of an RL system

Every RL problem is built from the same six building blocks. The grid-world robot — a robot navigating a room to find a door — is a helpful running example.

**Agent** — the learner or decision-maker. In the grid-world, the robot (or the software running inside it) is the agent. It observes the environment, selects actions, and receives feedback.

**Environment** — everything outside the agent. It generates observations, accepts the agent's actions, and returns a reward plus a new state. In the grid-world, the room (walls, floor, door) is the environment.

**State** — a formal representation of what the agent currently observes. In the grid-world, the state is the agent's (x, y) coordinate. In more complex settings, state could include sensor readings or camera data. Taking an action generally moves the agent to a new state.

**Action** — a choice the agent can make in a given state. In the grid-world, the four possible actions are: move left, move right, move up, move down. The complete set of possible actions is defined up front.

**Reward** — a numerical feedback signal the environment produces after the agent takes an action. The reward function takes the current state and the action as inputs and outputs a scalar. A high positive reward means the action was good; a negative reward means it was poor. The agent uses rewards to learn which actions to repeat and which to avoid.

**Policy (π)** — the function inside the agent that maps state observations to actions. It is the agent's strategy and the central output of RL training. The optimal policy is denoted **π*** (pi-star). Multiple policies may accomplish a task, but π* does so optimally — for example, in the fewest steps.

Driving all of this is the **learning algorithm** — the mechanism that updates the policy based on rewards. The two algorithms covered here are **Q-learning** (for value-based representations) and **policy gradient** (for policy-function-based representations).

---

### The RL interaction cycle

In reinforcement learning, the agent and environment communicate in a continuous loop:

```
observes state stakes action areturns reward R and new state s'updates policy via learning algorithmAgentEnvironment
```

Step by step:

1. The agent observes the current state of the environment.
2. The policy selects an action based on that state.
3. The environment transitions to a new state and emits a reward.
4. The agent receives the new state and the reward.
5. The learning algorithm updates the policy.
6. The cycle repeats until training converges.

### Policy representation: actor vs critic

When building an RL system, you must represent the policy mathematically. Two primary approaches exist.

**Policy-based (actor) approach** — the policy function maps states directly to actions:

```
π(state) → action

```

The policy is commonly represented as a **neural network** that receives state observations as input and outputs actions. Because this function is the sole decision-maker about what action to take, it is called an **actor**. The learning algorithm for this approach is **policy gradient**.

**Value-based (critic) approach** — instead of mapping states directly to actions, a value function maps (state, action) pairs to a scalar:

```
Q(state, action) → value

```

The value represents the long-term expected reward for being in a given state and taking a given action. A separate **max function** then selects the action with the highest value. Because this function analyses and "criticises" each possible action rather than selecting one directly, it is called a **critic**. The complete policy requires both the value function and the max function together.

A third option — the **actor-critic** method — combines both components into a single system.

**Watch out for:** the value function alone is not the policy. You also need the max function to turn Q-values into an action choice. Students often conflate the two.

---

### The Q-table

The **Q-table** is a concrete implementation of the value-based approach when states and actions are **discrete and finite**.

- Rows represent states.
- Columns represent actions.
- Each cell holds the **Q-value Q(s, a)**: the value of taking action a in state s.

What is known up front: the complete set of states and actions — these define the table's dimensions. What must be learned: the Q-value in every cell. All values start at zero.

For a 3×3 grid-world with 9 states and 4 possible actions, the Q-table has 9 × 4 = 36 cells, all initialised to 0.

**Limitation:** the Q-table only works when states and actions are discrete. Continuous states or actions cannot be represented in a table — that is when you need a neural network instead.

---

### Q-learning and the Bellman equation

**Q-learning** is the algorithm that populates the Q-table — it learns the optimal Q-value for every (state, action) pair.

The process runs in **episodes**. One episode ends when the agent reaches a terminal or goal state. Multiple episodes are run until Q-values stabilise.

Each Q-value update uses the **Bellman equation**:

```
Q_new(s, a) = Q_old(s, a) + α × [R(s, a) + γ × max_a' Q(s', a') − Q_old(s, a)]

```

Term | Meaning
Q_new(s, a) | Updated Q-value for state s, action a
Q_old(s, a) | Q-value before this update
R(s, a) | Immediate reward for taking action a in state s
s' | New state the agent transitions to
max_a' Q(s', a') | Highest Q-value achievable from the next state s'
γ(gamma) | Discount factor (0 ≤ γ ≤ 1) — weights future vs immediate rewards
α(alpha) | Learning rate — controls how much the new estimate replaces the old value

**Role of γ:**

- γ = 0: only the immediate reward matters; future rewards are ignored.
- γ close to 1: future rewards are weighted nearly as heavily as immediate rewards.
- 0 < γ < 1: future rewards are discounted — earlier rewards count more. This keeps the cumulative sum bounded.

**Role of α:**

- α = 1: the old Q-value is completely replaced by the new estimate.
- α < 1: the update blends old and new estimates.

**Worked example — 1-row grid with 3 states (S1, S2, S3):**

Setup: states S1, S2, S3 left to right; actions are left and right; every action gives a reward of −1; reaching S3 gives +10; α = 1, γ = 0.9; all Q-values start at 0.

- Episode 1, Step 1: agent starts at S1, randomly picks "right", receives reward −1. The Bellman equation updates Q(S1, right) to −1. Agent moves to S2.
- Episode 1, Step 2: agent is at S2, Q-values are equal so picks "right" at random, reaches S3. Q(S2, right) is updated to 10. Episode ends.
- After two episodes: Q-values stabilise. At S1, "right" has the higher Q-value, so the agent goes right to S2. At S2, Q(S2, right) = 10, so the agent goes right to S3.
- This is the **optimal policy** for this grid — the agent reaches the goal in the fewest steps without needing a map.

For larger environments, many more episodes are needed before Q-values stabilise.

### Markov decision processes (MDP)

A **Markov decision process (MDP)** is the general mathematical framework for modelling sequential decision-making problems. RL uses it to formally represent the problem so that algorithms can find an optimal solution.

MDP applies wherever outcomes are **partially random and partially controllable**, and a sequence of decisions must be made to reach a goal.

The relationship between MDP and RL is clean:

- **MDP** provides the formal representation (model) of the environment and the problem.
- **RL** provides the algorithms that learn the optimal policy within that representation.

An MDP has five components:

Component | Description
States (S) | The set of all possible states
Transition model (T) | Defines the next state when a given action is taken; can be deterministic or stochastic
Actions (A) | The set of possible actions; A(s) denotes actions available in state s
Reward (R) | Numerical signal received per state, state-action pair, or (s, a, s') triple
Policy (π) | The mapping from states to actions; optimal policy denoted π*

**The Markov property** is the key requirement for a decision process to qualify as an MDP. It states that the effect of an action depends **only on the current state, not on the prior history of states**. In the grid-world robot, the decision to move left or right depends only on the current cell coordinates, not on the path taken to get there.

**Stochastic vs deterministic transitions:** in a deterministic model, taking action a in state s always leads to the same next state s'. In a stochastic model, the same action leads to state s' with some probability p (for example, probability 0.7 to go right, 0.3 to go left). Stochastic models are common in real-world settings.

The goal of an MDP is to find π* — the policy that yields the **maximum cumulative discounted reward** over the agent's trajectory.

---

### Cumulative discounted reward

When a task requires multiple steps, the agent accumulates rewards over time. The total is not a simple sum — it uses the **discount factor γ**.

```
G_t = R_t + γ·R_{t+1} + γ²·R_{t+2} + γ³·R_{t+3} + …

```

Why does discounting matter? Without it, rewards at distant future steps contribute equally to the total, making the cumulative reward potentially unbounded. With 0 < γ < 1:

- Earlier rewards are weighted more heavily than later rewards.
- The discounted sum is bounded. When every reward equals a constant R, the sum converges to R / (1 − γ).

For example, applying γ = 0.9 to a multi-step reward sequence produces a finite, well-defined total — a concrete target for the agent to maximise.

---

### Policy gradient

**Policy gradient** is the learning algorithm used when the policy is a **neural network** (the actor/policy-based approach). It is the counterpart to Q-learning: Q-learning is used with value-based (Q-table) representations; policy gradient is used with policy-function-based representations.

Policy gradient directly optimises the policy by adjusting the neural-network weights **θ** to **maximise expected reward**. The objective function **J(θ)** is the expected cumulative reward. Because the goal is maximisation (not minimisation), the algorithm uses **gradient ascent** rather than gradient descent.

Update rule:

```
θ ← θ + α · ∇_θ J(θ)

```

- θ: current policy parameters (all neural-network weights)
- J(θ): objective function — expected cumulative reward
- ∇_θ J(θ): gradient of J with respect to θ
- α: learning rate

The positive sign (addition) moves θ in the direction that increases J. In contrast, gradient descent uses subtraction to minimise a loss function. Like gradient descent in supervised learning, policy gradient updates θ iteratively from an initial value until it converges on the θ* that maximises J.

**Comparing Q-learning and policy gradient:**

Aspect | Q-learning | Policy gradient
Policy representation | Value function (Q-table) | Policy function (neural network)
What is learned | Q-values for (state, action) pairs | Neural-network weights θ
Optimisation direction | Maximise Q-values via Bellman equation | Gradient ascent on expected reward J(θ)
Applicable when | States and actions are discrete and finite | States/actions can be continuous; policy is a neural network
Role of function | Critic (evaluates actions) | Actor (selects actions directly)

---

## 3. Key Takeaways

- Reinforcement learning is a trial-and-error paradigm where an agent learns by interacting with an environment over time — not from a fixed labelled dataset.
- Every RL problem has the same six components: agent, environment, state, action, reward, and policy (π). The goal is to learn the **optimal policy π*** that maximises cumulative reward.
- **Q-learning** populates a Q-table using the Bellman equation. The discount factor γ keeps future rewards bounded; the learning rate α controls how fast Q-values update.
- **Markov decision processes** formally represent sequential decision-making problems. The Markov property says that actions depend only on the current state — not on history.
- **Policy gradient** optimises a neural-network policy directly via gradient ascent on expected reward — use it when states or actions are continuous and a Q-table is impractical.

**Mental model:** Think of RL as training a new employee through experience — they try actions, receive feedback (rewards), and gradually learn the best strategy (optimal policy) without ever being handed an answer key.
