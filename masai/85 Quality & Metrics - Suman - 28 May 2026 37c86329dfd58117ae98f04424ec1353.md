## 2. Detailed Explanation

### Generative Adversarial Networks (GANs)

A **Generative Adversarial Network (GAN)** is a generative model that uses two competing components — a **Generator** and a **Discriminator** — to produce realistic images.

**Why it matters**

Before GANs, generating a convincing fake image from scratch was extremely difficult. The adversarial setup gives the generator a continuous feedback loop that pushes it toward realism without needing manually labelled data.

**Walkthrough**

The generator produces fake images. The discriminator tries to detect whether each image is real or fake. They play an adversarial game: every time the discriminator catches the generator, the generator learns from that mistake and improves. After many iterations, the generator becomes capable of producing images that fool the discriminator. Once training converges, you can use the generator to create new images on demand.

```
Caught fake → feedbackOutput verdictReal images (training set)Discriminator(real or fake?)Generator(produces fake images)Real / Fake decision
```

**Common mistakes**

- Assuming the discriminator is what you deploy — you only keep the generator after training.
- Expecting instant convergence; the adversarial game takes many iterations before the generator becomes convincing.

---

### Variational Autoencoders (VAEs)

A **Variational Autoencoder (VAE)** is a generative model that compresses images into a compact **latent space** and then reconstructs them from that compressed form.

**Why it matters**

A latent space is a compressed, continuous representation of images. Because it is continuous, you can sample from it or move through it smoothly — enabling controlled generation and interpolation between styles.

**Walkthrough**

The VAE has two parts. The **encoder** takes an input image and maps it to a position in latent space, described by a mean and a standard deviation (one distribution per image). The **decoder** takes a point sampled from that distribution and reconstructs an image. At inference time, you sample from the latent space and decode — producing a new image that follows the patterns the model learned.

This latent-space structure is also used inside Stable Diffusion: images are encoded into latent space, operated on there (much cheaper than pixel space), then decoded back to pixels.

**Common mistakes**

- Thinking the latent space stores the original image directly — it stores a *distribution* (mean + std), not a fixed point.
- Forgetting that the decoder reconstructs from a sample, so outputs vary slightly even from the same input.

---

### Diffusion Models

A **diffusion model** is a generative model that learns to reverse a gradual noise-addition process to produce clean images.

**Why it matters**

Diffusion models produce higher-quality and more diverse images than earlier approaches. **Stable Diffusion** is one of the most widely used implementations.

**Walkthrough**

Training happens in two passes:

1. **Forward pass** — noise is added to a real image step by step until the image is entirely noise.
2. **Reverse pass** — the model learns to denoise that image step by step, recreating a clean image.

The learning objective is simply to master the denoising process. At inference, you start from pure noise and run the reverse pass to produce a new image.

**Common mistakes**

- Confusing the forward pass (adds noise, deterministic, not learned) with the reverse pass (removes noise, learned by the model).
- Assuming fewer denoising steps always works well — quality degrades below a threshold.

### How Stable Diffusion Generates Images

**Stable Diffusion** runs a three-stage pipeline to convert a text prompt into a finished image.

**Why it matters**

Understanding the pipeline helps you know exactly which stage each parameter (prompt, CFG, steps) affects — and why changing even a single word in the prompt changes the output.

**Walkthrough**

The three stages are:

1. **CLIP text encoder** — converts the text prompt into numerical text embeddings that capture semantic meaning.
2. **U-Net denoiser** — starts from a noisy latent representation and denoises it iteratively (default: 25 steps). At each step it checks how close the current latent is to the text embedding and adjusts toward it.
3. **VAE decoder** — reconstructs the final image from the denoised latent.

Changing even one word shifts the embedding vector, which shifts the direction the U-Net steers toward — that is why prompt wording matters so much.

The key libraries are `diffusers` (pre-built pipelines and schedulers), `torch` (tensor computation and GPU management), `open_clip` (CLIP scoring), `matplotlib` (image display), and `numpy` (numerical operations). All these models require a GPU; CPU cannot run them at usable speed. **Attention slicing** is enabled when loading the model to reduce VRAM usage when GPU memory is limited.

**Common mistakes**

- Running these models on CPU — they will fail or be unusably slow.
- Forgetting to enable attention slicing on smaller GPUs, leading to out-of-memory errors.

### Prompt Engineering for Image Generation

**Prompt engineering** is the practice of designing text inputs to reliably produce the image you want from a generative model. A vague prompt produces a vague image; a precise prompt produces a precise image.

**Why it matters**

The CLIP text encoder converts every word into an embedding vector. Each component of the prompt contributes its own signal. Leaving components out means the U-Net has nothing to steer toward for that aspect of the image.

**Walkthrough**

A useful starting point is a **baseline** — generating images with minimal prompts (subject and clothing only, no style modifiers). The baseline shows the least possible quality when the model is used as-is. All subsequent experiments are measured against it, exactly as a baseline functions in any data science experiment.

The recommended prompt template has six components:

```
{subject}, {clothing}, {style}, {lighting}, {camera}, {quality}

```

Component | Example values
Subject | "fashion model", "young woman"
Clothing | "wearing emerald green silk blouse and tailored black trousers"
Style | "editorial fashion photography, Vogue Max"
Lighting | "softbox studio light", "dramatic rim light", "golden hour sun"
Camera | "85 mm portrait lens", "close up", "full body"
Quality | "ultra-realistic", "8K", "sharp focus"

Build prompts iteratively: start with subject only, then add clothing, then style, then lighting, then camera, then quality. Each addition produces a visibly more refined image.

**Three failure modes to know:**

- **Weak prompt** — "a person wearing clothes" produces a generic, low-quality result with no aesthetic intent.
- **Contradictory modifiers** — mixing incompatible terms (e.g., "summer dress, winter snow, beach, dark moody, bright, cheerful, indoor, outdoor") forces the U-Net to attend to conflicting embedding signals. The output becomes incoherent. Fix: be specific and internally consistent.
- **Negatives in the positive prompt** — writing "image shouldn't be blur" in the positive prompt causes blurry output because the model tends to hallucinate the negated feature. Fix: always use the dedicated `negative_prompt` parameter.

**Production-ready settings:**

- CFG: 7.5
- Denoising steps: 30
- Negative prompt: `blurry, low quality, ugly, bad anatomy, watermark, distorted, artifacts`

**Common mistakes**

- Putting avoidance terms inside the positive prompt text — the model does the opposite.
- Using contradictory descriptors (bright + dark, indoor + outdoor) without realising they produce conflicting embedding signals.

### Classifier-Free Guidance (CFG)

**CFG** (Classifier-Free Guidance), also called guidance scale, controls how strictly the model follows the prompt versus exercising its own creativity.

**Why it matters**

CFG is one of the most important parameters in Stable Diffusion for controlled generation. Too low and the model ignores your prompt; too high and the image looks artificial.

**Walkthrough**

CFG range | Effect
1–3 (low) | Highly creative; may ignore parts of the prompt
7–8 (sweet spot) | Best balance of creativity and prompt accuracy
15+ (high) | Oversaturated, unnatural artifacts; clearly machine-generated appearance

The sweet spot for most fashion generation tasks is CFG 7–8, with 7.5 as the recommended production value.

**Common mistakes**

- Setting CFG very high thinking it will make the model "follow the prompt better" — above ~15 it introduces artifacts and oversaturation instead.

### Negative Prompts

**Negative prompts** tell Stable Diffusion what to avoid in the generated image. They are passed as a separate `negative_prompt` parameter (abbreviated `neg` in code), not embedded in the positive prompt text.

**Why it matters**

Without negative prompts, the model may produce blurry, distorted, or stylistically wrong output that the positive prompt alone cannot prevent.

**Walkthrough**

Four levels of negative prompting were tested:

1. **No negatives** — model produces arbitrary, sometimes blurry or distorted output.
2. **Basic quality** (`blurry`, `low quality`) — visible improvement.
3. **Full filter** (`blurry`, `low quality`, `ugly`, `bad anatomy`, `watermark`, `distorted`, `artifacts`) — significantly cleaner output.
4. **Style + quality** — full filter plus `cartoon`, `illustration`, `painting` — most polished result.

Common negative-prompt terms: `blurry`, `low quality`, `ugly`, `bad anatomy`, `distorted`, `artifacts`, `watermark`, `cartoon`, `illustration`, `painting`.

**Common mistakes**

- Placing negative concepts inside the positive prompt (e.g., "image shouldn't be blur") — the model hallucinates the negated feature instead.

---

### VAE Latent-Space Interpolation

**Latent-space interpolation** is the process of smoothly blending two images by moving between their latent-space representations step by step.

**Why it matters**

Interpolation demonstrates that the VAE latent space is continuous and meaningful — the model can navigate between two distinct visual styles by moving through the compressed representation, rather than making hard cuts.

**Walkthrough**

Two fashion images are generated:

- Image A: "elegant black evening gown, haute couture, studio lighting, full body"
- Image B: "oversized hoodie"

Each image is encoded into the VAE latent space (described by its mean and standard deviation). Interpolation is then performed between the two latent representations across eight steps. At each step, the decoder reconstructs an image. The result is a smooth visual transition from the evening gown to the hoodie, passing through intermediate blended styles.

The transformation pipeline for each image before encoding: convert to tensor → normalize (albumentations normalization) → unsqueeze → pass through VAE encoder.

**Common mistakes**

- Expecting a sharp cut between the two styles — the output is a gradual blend because the latent space is continuous.

### Fine-Tuning: LoRA

**LoRA (Low-Rank Adaptation)** is a parameter-efficient fine-tuning technique that adapts a pre-trained model by adding small trainable adapter matrices while keeping the original weights frozen.

**Why it matters**

Stable Diffusion has 860 million parameters. Full fine-tuning updates all of them, which is expensive and risks **catastrophic forgetting** — the model loses knowledge from its original training. LoRA trains roughly 1% of the parameters instead, making it fast, cheap, and safe. The same technique works for both LLMs and vision models like Stable Diffusion.

Prompting can only instruct the model; it cannot give the model a skill it was never trained on. For enterprise-specific style or private data, fine-tuning is necessary.

**Walkthrough**

The LoRA mechanism:

1. Freeze the original weight matrix (e.g., 768 × 768).
2. Inject two small adapter matrices **A** (N × R) and **B** (R × N), where R is the chosen rank.
3. Train only A and B. Their product A × B reconstructs a matrix of the original shape (768 × 768).

At rank R = 4:

- Matrix A: 768 × 4
- Matrix B: 4 × 768
- Trainable parameters: 768 × 4 + 4 × 768 = **6,144** instead of 768 × 768 = **589,824**
- This is approximately **1% of the original parameters**.

Rank choices and trade-offs:

Rank (R) | Adapter size | Expressiveness
4 | Very small | Minimal, fast to train
8 | Small | Good for subtle style changes
16 | Medium | More expressive
32–128 | Larger | Approaching full fine-tuning cost

**LoRA teaches style or aesthetic** — it adapts the model to produce outputs in a consistent visual style.

LoRA adapters can be trained from scratch by freezing base layers, adding adapter layers, training on custom data, and saving or pushing to Hugging Face. Alternatively, pre-trained adapters can be downloaded from Hugging Face and loaded directly onto the base model.

**Common mistakes**

- Choosing rank 128 thinking more is always better — higher rank increases cost and narrows the gap from full fine-tuning.
- Using LoRA when you need to teach a specific *object or person identity* — that is DreamBooth's job.

### Fine-Tuning: DreamBooth

**DreamBooth** fine-tunes a model to learn a specific subject or object — not just a style — by binding it to a rare token.

**Why it matters**

LoRA changes the aesthetic. DreamBooth changes the subject. For use cases like standardising customer photos into passport-size style, or generating images of a specific branded product in any scene, DreamBooth is the right tool.

**Walkthrough**

The DreamBooth mechanism:

1. Select 5 or more photos of the specific product or person, each with a caption.
2. Use a **rare token** (e.g., `SKS`) — a word the model has never seen during original training. Because it has no prior association, the model can be trained to link `SKS` exclusively to the target subject.
3. After training, any prompt containing that token produces the target subject regardless of background, setting, or action.

Example: prompts "SKS fashion dress on model", "SKS fashion dress in Paris", and "SKS fashion dress editorial" all produce images consistent with the trained dress, varying only context.

Another example: a model trained on a cartoon or Ghibli-like art style will produce output in that same artistic style regardless of the background or scene described in the prompt. This demonstrates how DreamBooth locks the underlying aesthetic while allowing the context to change.

**Key distinction:**

- LoRA teaches a style (e.g., cinematic lighting, editorial photography aesthetic).
- DreamBooth teaches a specific subject or object identity (e.g., a particular product, a particular person's face).

**Common mistakes**

- Using a common word as the token — the model already has associations for common words, so it cannot bind the new concept cleanly. Use a rare or invented token like `SKS`.

### FID — Fréchet Inception Distance

**FID (Fréchet Inception Distance)** measures how similar the statistical distribution of generated images is to the distribution of real images. Lower FID means more similar distributions.

**Why it matters**

A single generated image can look convincing by luck. FID checks the entire distribution — whether the model produces a realistic variety of images consistently, not just occasionally.

**Walkthrough**

How FID is calculated:

1. Pass all real images through Inception V3 — extract feature vectors (2,048 features per image).
2. Pass all generated images through Inception V3 — extract feature vectors.
3. Fit a Gaussian distribution to each set of feature vectors.
4. Calculate the Fréchet distance between the two Gaussians.

Interpretation:

FID value | Meaning
0 | Generated images are statistically identical to real images
< 10 | Excellent
< 30 | Good
> 50 | Poor quality generation

**Important limitation:** A high FID could mean the generated images are bad, or it could mean they are stylistically different from the reference set in a deliberate and valid way. FID does not distinguish between these two cases. FID also requires actual real reference images — without them, the metric is unreliable.

**PCA visualization:** The 2,048 features per image cannot be plotted directly. Principal Component Analysis (PCA) reduces them to 2 principal components so you can visualize and compare the feature clusters of different models. Clusters that are close together have lower FID.

**Common mistakes**

- Treating a high FID as always meaning bad quality — it could simply mean the generated style differs from the reference set intentionally.
- Trying to compute FID without a set of real reference images.

---

### CLIP Score

**CLIP score** measures text–image alignment — whether a generated image actually reflects the content described in the prompt.

**Why it matters**

FID tells you whether the distribution looks real, but not whether the model is following your prompt. CLIP score fills this gap. It also does not need real reference images, making it cheaper to compute.

**Walkthrough**

How CLIP score is calculated:

1. Pass the text prompt through the CLIP text encoder → text embedding (a vector).
2. Pass the generated image through the CLIP image encoder → image embedding (a vector).
3. Compute cosine similarity between the two vectors.

Higher CLIP score = better alignment between prompt and image. If the score drops below a threshold, the model is not following the prompt.

**FID vs. CLIP score:**

Metric | What it measures | Requires real images? | Higher = better?
FID | Distribution similarity (generated vs. real) | Yes | No (lower is better)
CLIP score | Text–image alignment | No | Yes

**Common mistakes**

- Assuming a high CLIP score means the image looks photorealistic — it only means the image matches the text, not that it looks like a real photo.

### Using Metrics in a Production Pipeline

**Why it matters**

No single metric tells the full story. Combining FID, CLIP score, and human evaluation gives a complete picture of generative model quality.

**Walkthrough**

A practical production pipeline:

- Store a set of real reference images collected during model training.
- Periodically run FID by comparing those reference images against new batches of generated images to detect distribution drift.
- Run CLIP score on every generated image to verify prompt adherence.
- Use selective human evaluation to assess aesthetic appropriateness and real-world usability — the aspects that automated metrics cannot fully capture.

Metrics are calculated programmatically; libraries handle feature extraction and scoring automatically. Human sampling supplements the automated scores.

**Common mistakes**

- Relying on FID alone and missing prompt-following failures (low CLIP score despite good FID).
- Skipping human evaluation entirely — machines cannot always detect aesthetic problems that are immediately obvious to a client.

## 3. Key Takeaways

- GANs use a generator–discriminator adversarial game, VAEs compress images into a continuous latent space, and diffusion models learn to reverse a noise-addition process — three distinct approaches to image generation.
- Stable Diffusion chains a CLIP text encoder → U-Net denoiser (25 steps default) → VAE decoder; every prompt word shifts the embedding and therefore shifts the generated image.
- Prompt engineering has a clear template (subject, clothing, style, lighting, camera, quality); the three failure modes to avoid are weak prompts, contradictory modifiers, and placing negatives inside the positive prompt.
- LoRA fine-tunes style with ~1% of parameters (rank-4 adapter = 6,144 trainable parameters vs. 589,824 original); DreamBooth fine-tunes a specific subject identity using a rare token and 5+ reference photos.
- FID measures distribution similarity to real images (lower is better, requires reference images); CLIP score measures text–image alignment (higher is better, no reference images needed); human evaluation adds real-world judgment that neither metric captures alone.

**Mental model:** Building a generative image system involves three layers. The base model knows how to generate (GAN/VAE/diffusion). The control layer steers what gets generated (prompt engineering + CFG + negatives). The quality layer measures how well it worked (FID + CLIP + human review).
